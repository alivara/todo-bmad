## Epic Summary

Epic 1 built the running, demoable **walking skeleton** — a greenfield monorepo (`web`/`api`/`shared`), a fixed wire contract, a design-token system, migrations-on-boot, and one-command `docker compose up` — then delivered optimistic task **create** and **view** against durable Postgres, with a minimal resilience floor.

Every story passed full gates (eslint/prettier + gofmt/vet/golangci-lint + unit) and — from 1.2 on — live `docker compose up` + browser verification. Every story went through adversarial review, most through multiple passes.

## What Went Well

- **Foundation patterns held for the entire project.** Three seams established here were *extended*, not reinvented, by later epics:
  - **AD-6 shared wire contract + drift guard** — `shared/todo.ts` as single source of truth; 1.1 seeded it, 1.2 turned on enforcement (`contract_test.go` pins the JSON key set/nesting → red build on drift, plus Playwright `assertTodoShape`).
  - **Serializer-owns-wire-shape** — `api/handler/wire.go` produces the AD-6 shape; the domain `Todo` stays flat `time.Time`. 1.2 reused `toTodoResponse` rather than hand-building JSON.
  - **Optimistic-update + rollback hook** (`useCreateTodo`, 1.2) — cancel→snapshot→prepend temp row→per-temp-id rollback→swap temp id for server UUID→reconcile. Epic 2's edit/toggle/delete and Epic 3's R1 systematized wrapper build on this exact shape.
- **The testseed reset seam (1.1/1.2) became Epic 4's substrate.** Build-tag-guarded, prod-unreachable (`testhelpers/seed.go` + `POST /internal/test/reset` + `docker-compose.test.yml`). Test hooks built before CI existed paid off two epics later in the integration/E2E lane.
- **Adversarial review earned its keep from story one — real defects, not nits.** golangci-lint was genuinely missing in 1.1 (the Go quality gate wasn't actually met at dev-done); 1.2 had an optimistic-rollback **concurrency clobber** (whole-list restore vs per-temp-id restore) plus a missing AC9 injection test; 1.3 shipped a clamp that guessed 2-line overflow by string length instead of measuring the DOM, and a ~375px mobile layout break.
- **Loopback quality improved across the epic.** Story 1.3 (spec-kernel format) recorded **0 intent-gaps and 0 bad-spec loopbacks** across both passes — review found code defects, not misunderstood requirements.
- **Disciplined scope boundaries.** 1.3 chose *not* to render a dead checkbox for completed status (Epic 2 owns completion), resolving five under-specifications conservatively and flagging them for ratification rather than scope-creeping.

## What Was Harder / Lessons Learned

- **E2E tests were written but not runnable locally** — 1.2 and 1.3 both leaned on "verified at the CI gate" that did not yet exist. This was Epic 1's biggest gap, and it is precisely the problem **Epic 4 (CI lanes) later solved** — a need Epic 1 identified and a later epic fulfilled.
- **The dumb proxy kept springing edge-case leaks** — client-abort handling, timeouts, hop-by-hop headers (1.1), and the 204/304 null-body throw in `new Response()` (1.2, which would also have bitten Epic 2's DELETE). `web/app/api/[...path]/route.ts` was a repeat offender across stories.
- **Accessibility contrast was a recurring theme rooted in a day-one choice.** The Cream & Terracotta palette was locked in 1.1, but it carries a latent below-AA contrast defect (`--ink-secondary` #8a8072 on #fffcf7 ≈ 3.8:1; completed-row `--ink-muted` ≈ 1.9:1). Every story that rendered text re-surfaced it (1.2 focus/aria deferrals, 1.3 body-copy + completed-row contrast), and a later axe audit confirmed it and added the "Add" button. All routed to Story 3.5.
- **Dev-time "done" over-claimed more than once** — 1.1 declared the Go quality gate satisfied while golangci-lint wasn't committed/run. Review caught it, but "done" should mean the gate actually ran green.
- **Go toolchain absent on host** — every Go command ran inside `golang:1.26-alpine` / a golangci-lint container for the whole epic. Managed (documented as "previous-story intelligence"), but persistent friction.

## Key Insights

1. Invest in foundation seams early — the wire contract, the optimistic hook, and the testseed reset seam compounded across every later epic.
2. Adversarial review reliably catches over-claimed "done" (missing linter, concurrency bug, wrong clamp heuristic) — keep it mandatory.
3. Locally-unrunnable E2E was the foundation's biggest gap; sequencing CI (Epic 4) next was the right call.
4. **Verify WCAG AA contrast when design tokens are defined, not at first render** — a day-one palette choice seeded a cross-cutting a11y debt that surfaced story after story. Catch it at token-definition time.
