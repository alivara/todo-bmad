## What Was Harder / Lessons Learned

- **Optimistic-state correctness under concurrency/refetch was the recurring hazard.**
  - 2.1: same-tick double-tap desync — target status read from a stale `todo` prop; fixed by reading current status from the query cache.
  - 2.3 (HIGH): the pending-delete controller originally modeled "deleted" as *absence* from the cache. The server still holds the row during the ~5s undo window, so any refetch (a concurrent toggle's `onSettled` invalidate, or window-focus refetch) **resurrects it** → dup React key on undo / zombie row on elapse; reachable via "delete one task, toggle another." Re-architected to a **suppressed-id set + list `select` filter** (cache holds server truth, mutated only on commit-success; undo just un-suppresses).
- **Test config that diverges from prod masked the HIGH bug.** 2.3's tests used `staleTime: Infinity`, so no refetch ever fired — the resurrection couldn't reproduce in the suite. Only caught because the reviewer reworked the harness to mirror prod (`staleTime: 5000`, `select` filter). Lesson: mirror prod config or the test hides the exact class of bug it should catch.
- **The trickiest branch shipped untested until review forced it — every story.** 2.1's dynamic `$n` UPDATE builder + id-scoped rollback; 2.2's editor-scoped blur (tab title→description must not save); 2.3's keepalive/concurrency. Lesson: write the crux branch's tests proactively.
- **Every new interactive control needed a keyboard-reachability fix in review.** 2.1 real `<button role=checkbox>`; 2.2 tap-to-edit was mouse-only (keyboard users couldn't enter edit mode); 2.3 focusable ✕/Undo. Deeper a11y (toast `aria-live`, full toast keyboard traversal, contrast) was deliberately deferred to the Epic 3 floor.
- **Rollback isn't done without a visible, scoped, auto-clearing error** — 2.1 shipped a rejected toggle with *no* error surface (AC3 gap); 2.3 had an error that clobbered a newer toast's Undo and was sticky. Both fixed in review.
- **A process-honesty flag:** 2.3's mechanism was *materially wrong* (the absence model) yet recorded `bad_spec: 0`, patched via a Spec Change Log rather than a spec re-derive. Defensible (backend/UI/race-guard were all correct), but worth watching that bad_spec isn't under-counted.
- **`sendBeacon` doesn't fit DELETE** — it's POST-only; 2.3 correctly switched the unload flush to `fetch(..., { method: 'DELETE', keepalive: true })` on `pagehide` (bfcache-safe). The epics.md AC still says "sendBeacon" — a doc/plan detail that reality corrected.

## Key Insights

1. Optimistic "absence" is unsafe while server truth persists — model client-only state as a suppression overlay, and read the cache, not props.
2. Test config must mirror prod (`staleTime`, refetch) or it hides the very bug class you're testing.
3. Write the crux/riskiest branch's tests proactively; don't rely on review to add them.
4. Build keyboard-reachability + labeling into every new control up front.
