## 2. What Went Well

- **The expert-reviewer swap paid for itself.** Gopher caught the one HIGH of the epic (3.1: `gin.Recovery()` emitted a bare bodyless 500 that escaped the AD-9 error envelope → fixed with `gin.CustomRecovery`). Pixel caught a string of a11y/UX defects a generic reviewer would likely have missed (focus-ring contrast, the cold-scrollbar `color-scheme` gap, the mid-transition axe scan reasoning).
- **Investigation-first specs.** Pre-computing the hard stuff before writing code — WCAG ratios for *both* themes (3.5), the no-flash hydration seam (3.4), the CSP-vs-inline-script interaction (3.5) — meant the implementation subagents had unambiguous targets and the specs held up under adversarial review.
- **The integration-e2e lane is a real safety net.** It caught what local Vitest structurally cannot: strict-mode locator traps (3.1) and a live-axe contrast failure (3.5). Every red was pre-merge.
- **Contrast fix landed cleanly in both themes** with a minimal 4-token change and a completed-text reroute — verified by independent recomputation in review.

## 3. Challenges & Patterns

- **E2E-only failure modes bit twice.** (a) 3.1's optimistic-rollback spec hit Playwright strict-mode locator traps (`getByRole('alert')` matching Next's route-announcer; split text nodes) across two CI rounds. (b) 3.5's new hard contrast gate went red because `a11y-005` toggled to dark then scanned **mid-transition** — axe sampled the card's 350ms background fade at ~83% (`#514c47`) under dark text. Neither is reproducible in unit tests; both needed the live lane (and, for 3.5, a local route-stub repro) to diagnose. **The settled palette was always correct — the bug was test timing, not the design.**
- **A sibling test encoded a mechanism 3.5 replaced.** `2.1-E2E-003` asserted the completed card's old `opacity:0.85` recede, which 3.5 swapped for a chrome recede (transparent surface, full-opacity text) to keep the text AA. The test had to move with the mechanism.
- **Deferred-ledger vs epic-AC scope fork (3.5).** `deferred-work.md` routed contrast, edit-discoverability, and security hardening *to* 3.5, but the epic ACs named only a subset (and allowed "verified ratios" to be deferred). The right call was to surface the fork for an explicit decision rather than guess — the stop-and-ask worked.
- **Mid-flight parallel merge.** PR #27 (optional create-description field) merged to main during 3.5, adding a new `<textarea>` with the same focus-halo gap it routed to 3.5. Resolved the conflict and extended the `.add-input` focus fix to it rather than leave a fresh gap.
